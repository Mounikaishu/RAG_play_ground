GOLDMAN SACHS SOFTWARE ENGINEER INTERVIEW EXPERIENCE
Company: Goldman Sachs
Role: Software Engineer
Difficulty: Hard

HACKERRANK TEST (Round 1)
- Duration: 120 minutes
- Section 1: 5 MCQs on Operating Systems, DBMS, and Networking
  - OS: Process scheduling, deadlocks, virtual memory paging
  - DBMS: ACID properties, normalization (up to BCNF), indexing strategies
  - Networking: TCP vs UDP, HTTP/HTTPS, DNS resolution
- Section 2: 2 coding questions
  - Question 1 (DP): Minimum cost to reach the last cell in a grid
  - Question 2 (Strings): Longest substring with at most K distinct characters

TECHNICAL ROUND 1 (Round 2)
- Duration: 60 minutes
- Theory: Explain ACID properties with real-world examples
  - Deep dive into isolation levels (Read Uncommitted, Read Committed, Repeatable Read, Serializable)
- Design: Design a Stock Trading Platform
  - Order matching engine, real-time price updates
  - Discussed eventual consistency vs strong consistency
  - Database sharding strategies for trade history
- Coding: Implement merge sort and explain its time complexity
  - Discussed stability of sorting algorithms and when to use which

TECHNICAL ROUND 2 (Round 3)
- Duration: 60 minutes
- Java threading concepts: synchronized blocks, volatile, ThreadPool
- Coding: Producer-Consumer problem using Java concurrency
- Database: Complex SQL queries involving window functions, CTEs
  - Write a query to find the second-highest salary per department
  - Running average of stock prices using window functions
- Discussion on microservices vs monolithic architecture trade-offs

HR + MANAGERIAL ROUND (Round 4)
- Why Goldman Sachs? Research their technology division
- Ethical scenario: What would you do if you found a bug that benefits the company but hurts clients?
- Salary expectations and joining timeline
- Questions about work-life balance and team culture

TIPS FOR PREPARATION
1. Goldman Sachs tests CS fundamentals deeply — revise OS, DBMS, and networking concepts
2. Practice SQL with window functions, CTEs, and complex joins
3. Strong Java/C++ knowledge is essential — especially threading and concurrency
4. System design for financial systems: focus on consistency, reliability, and audit trails
5. Be prepared for ethical questions — Goldman values integrity
6. Practice competitive programming for the HackerRank test (DP and string problems are common)
